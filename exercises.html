2.1 Esercizio 1

Si scriva in un qualsiasi pseudo linguaggio un frammento di codice tale che il numero massimo dei RdA presentia run-time sulla pila non sia determinabile statisticamente.

E' sufficiente mostrare un esempio di ricorsione 

int fact(int val){
  if(val == 0)
    return 1;
  else
    return fact(val - 1) * val;
}

printf("Results : %d", fact(10)); // 10 puo essere un valore preso in input

Results : 3628800

2.2 Esercizio 2

Si scriva in un qualsiasi pseudolinguaggio una funzione ricorsiva tale che il numero massimo dei RdA per essa presenti a run-time sia determinabile statisticamente Si puo' generalizzare?

Riproponoiamo l'esempio di prima senza ricorsione 

int fact(int val){
  int out = 1; 
  for(int i = 1; i < val; i ++)
    out *= i;
  return out;
}

printf("Results : %d", fact(10)); // 10 puo essere un valore preso in input

Results : 362880

Si puo' genralizzare escludendo la ricorsione dal nostro linguaggio. Ogni procedura necessitera' di una sola area di memoria in quanto non vi potreanno essere chiamate conteporanee di quest'ultima.

2.3 Esercizio 3

Consideriamo il seguente frammento di pseudo-codice 

A:{int X = 1;
	 ...
	 B:{ X = 3;
		 ...
		 }
	 ...
	}

Supponiamo il B si trovi annidato di un livello rispetto ad A. Perche' per risolvere il riferimento a X presente in B non basta considerare il record di attivazione che sulla pila precede immediatamente quello di B? Si fornisca un controesempio riepiendo gli spazi indicati con i puntini con del codice opportuno

Non e' possibile risolvere il riferimento come suggerito perche' il fatto che due blocchi siano annidati non significa che la sequenza di chiamata delle procedure (corrispondenti ai blocchi) corrisponda alla strutturazione del programma. E' sufficiente ad esempio che vi sia un altro blocco parallelo a B chiamato prima di quest'ultimo per far si che sulla pila il record di attivazione precedente a B sia lo stesso C. Nell'esempio la risposta dovrebbe risultare piu chiara. 

A:{int X = 1;
	 void pippo(){
		 B:{ X = 3;
			 ...
			 }
	 }
	 C:{
		 Y = 5;
		 pippo();
	 }

2.5 Esercizio 6

E' piu' semplice da implementare la regola di scope statico o quella di scope dinamico? Si motivi la risposta

E' piu' facile da implmentare la regola dello scope dinamico in quanto e' sufficiente sfruttare la architettura a pila della memoria per trovare la prima occorrenza/definizione del nome non locale. (Sono comunque implementate tecniche per velocizzare la ricerca dei nomi come ad esempio CRT) La gestione dello scope statico dipende dalla struttura del programma e non dalla sua esecuzione pertanto a run-time e' necessario gestire e nel prologo e nel epilogo del chiamato la catena statica.
