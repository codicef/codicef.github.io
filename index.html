3 Basic Notation

3.1 General

  • Superscript (i) will denote the \(i^{th}\) training example while superscript [l] will denote the \(l^{th}\) layer

3.2 Sizes

  • m : number of examples in the dataset
  • \(n_x\) : input size
  • \(n_y\) : output size (number of classes)
  • \(n_h^{[l]}\) : number of hidden units of the \(l^{th}\) layer
  • L : number of layers in a network

3.3 Objects

  • \(X \in R^{n_x \times m}\) : input matrix (samples in column for nn implementation purposes) \[X = \begin{bmatrix}& & & \\ x^{(1)} & x^{(2)} & ... & x^{m}\\ & & & \end{bmatrix}\]
  • \(x^{(i)} \in R^{n_x}\) : \(i^{th}\) example as column vector
  • \(Y \in R^{n_y \times m}\) : output/label matrix \[Y = \begin{bmatrix}& & & \\ y^{(1)} & y^{(2)} & ... & y^{m}\\ & & & \end{bmatrix}\]
  • \(y^{(i)}\) : output for the $ith example
  • \(W^{[l]} \in R^{\text{number of units in next layer} \times \text{number of units in the previous layer}}\) : weight matrix, superscript [l] indicates the layer
  • \(b^{[l]} \in R^{\text{number of units in the previous layer}}\) : bias vector in the \(l^{th}\) layer
  • \(\hat{y} \in R^{n_y}\) is the predicted output vector.

4 Logistic Regression

Logistic regression is a learning algorithm used in a supervised learning problem when the output y are all either zero or one. The goal of logistic regression is to minimize the error between its predictions and training data.

4.1 Sigma function

\(\sigma(z) = \frac{1}{1 + e^{-x}}\)

  • if z is a large positive number, then \(\sigma(z) = 1\)
  • if z is a large negative number, then \(\sigma(z) = 0\)
  • if z = 0 , then \(\sigma(z) = 0.5\)

Python version 

s = 1 / (1 + np.exp(- z))

4.2 Hyphotesis Function

Given x, \(\hat{y} = P(y = 1 | x)\) where \(0 \leq \hat{y} \leq 1\)

\(z = w^T x + b\) \(a^i = \hat{y} = \sigma (z) = \sigma (w^T x + b)\)

Vectorized in Python 

A = sigmoid(np.dot(w.T, X) + b)

4.3 Cost Function

\[J(w, b) = \dfrac{1}{m} \sum_{i=1}^m \mathrm{Loss}(\hat{y},y^{(i)})\]

Our loss function will be

\(\mathrm{Loss}(\hat{y}^{(i)},y^{(i)}) = -\log(\hat{y}^{(i)}) \:\: \text{if} \: y^{(i)} = 1}\) \(\mathrm{Loss}(\hat{y}^{(i)},y^{(i)}) = -\log(1- \hat{y}^{(i)}) \:\: \:\: \text{if} \: y^{(i)} = 1}\)

We can simplify everything in one single formula

\[\mathrm{Loss}(\hat{y}^{(i)},y^{(i)}) = - y^{(i)} \; \log(\hat{y}^{(i)}) - (1 - y^{(i)}) \log(1 - \hat{y^{(i)}})\]

J Loss function graphs

Cost function \[J = - \frac{1}{m} \displaystyle \sum_{i=1}^m [y^{(i)}\log (a^{(i)} (x^{(i)})) + (1 - y^{(i)})\log (1 - a^{(i)})]\]

Python version 

cost = -(1 / m) * np.sum(Y * np.log(A) + (1 - Y) * np.log(1 - A))

4.4 Gradient

\(dz^{(i)} = a^{(i)} - y^{(i)}\) \(dz = \frac{1}{m} \sum_{i = 1}^m a^{(i)} - y^{(i)}\)

dw \[dw^{(i)}_j = \sum_{i = 1}^m x^{(i)}_j dz^{(i)}_j =\sum_{i = 1}^m x^{(i)}_j (a^{(i)} - y^{(i)})\]

\(dw = \frac{1}{m} \: X \times (A - Y)^T\)

db \(db^(i) = a^{(i)} - y^{(i)}\)

\(db =\frac{1}{m} \sum_{i = 1}^m a^{(i)} - y^{(i)}\)

Python version 

dz = A - Y

dw = (1 / m) * np.dot(X, (A - Y).T)

db = (1 / m) * np.sum(dz)

4.5 Gradient Descent Algorithm

Repeat{ \[w_j := w_j - \frac{\alpha}{m} \sum_{i=1}^m (\hat{y}^{(i)} - y^{(i)}) x_j^{(i)}\] }

Python Vectorized 

w = w - learning_rate * dw
b = b - learning_rate * db

5.5 Gradient descent

5.5.1 Algorithm

Repeat{

  • compute predicts \(a^{[L](i)}, \: \: i=1,\dots, m\)
  • for each level l
    • \(dw^{[l]} = \frac{dJ}{dw^{[l]}}\)
    • \(db^{[l]} = \frac{dJ}{db^{[l]}}\)
    • \(w^{[l]} := w^{[l]} - \alpha dw^{[l]}\)
    • \(b^{[l]} := b^{[l]} - \alpha db^{[l]}\)

}

for l = L to 1

  • \(dz^{[l]} = A^{[l]} - Y\)
  • \(dw^{[l]} = \frac{1}{m} dz^{[l]} \times A^{[l- 1]T}\)
  • \(db^{[l]} = \frac{1}{m} \sum (dz^{[l]})\)

5.6 Backward propagation

\(da^{[l]}\)
Output
\(da^{[l - 1]}, \: dW^{[l]}, \: db^{[l]}\)

for each layer l

5.7 Random Initialization

For logistic regression it was okay to initialize weights to zero, but using neural networks we have to initialize them to random values.

If we initialize all weights to zero the hidden units will compute the same function (simmetric) and that is a problem because they become useles.

The solution is to initialize each level to \(W^{[l]} = \text{np.random.radn}(n_l, n_{l-1}) * 0.01\). It is important to multiply the random values to a small number because if it is too large gradient will be very small.

(If you have heard of "Xavier initialization", this is similar except Xavier initialization uses a scaling factor for the weights \(W[l]\) of sqrt(1./layersdims[l-1]) where He initialization would use sqrt(2./layersdims[l-1]).)

6.3 Bias and variance

When we are evaluating an error we should set ideally a base error to the error that even an human expert in the problem's subject would have.

6.3.1 High bias

High bias : underfitting

  • both \(J_{train}(\Theta)\) and \(J_{CV}(\Theta)\) will be high. Also, \(J_{CV}(\Theta) \simeq J_{train}(\Theta)\)

The first problem to solve in our problem structure is high bias

Solutions

  • Try a larger Neural Network (usually it does not affect bias/variance tradeoff)
  • Improve our dataset
  • Try to train longer
  • Improve Neural Network structure

6.3.2 High variance

High variance : overfitting

  • \(J_{train}(\Theta)\) will be low and \(J_{CV}(\Theta)\) will be much greater than \(J_{train}(\Theta)\).

Once we solved bias problems, we should fix high variance problems Solutions

  • More data (usually it does not affect bias/variance tradeoff)
  • Regularization
  • Improve Neural Network structure

6.6 Normalization

normalization of ratings means adjusting values measured on different scales to a notionally common scale, often prior to averaging. In more complicated cases, normalization may refer to more sophisticated adjustments where the intention is to bring the entire probability distributions of adjusted values into alignment. In the case of normalization of scores in educational assessment, there may be an intention to align distributions to a normal distribution.

Steps for normalization

  1. Calculate \(\mu = \frac{1}{m} \sum_{i = 1}^m x^{(i)}\)
  2. Calculate variance \(\sigma = \frac{1}{m} \sum_{i = 1}^m x^{2_{(i)}}\)
  3. To normalize for each \(x^{(i)}\), do \(x^{(i)} = \frac{x^{(i)} - \mu}{\sigma}\)

6.6.1 Goal of normalization

The goal of normalization is to speed up the gradient descent process because all the features would have the same range of values (higher gradients).

6.7 Gradient checking

6.7.1 Derivative approximation

The simples method to get an approximation of a derivative value is to use the definition

\(g(\Theta) = \frac{f(\Theta + \epsilon) -f(\Theta - \epsilon) }{2 \epsilon}\)

Error approximation :: \(O(\epsilon^3)\)

6.7.2 Implemetation

  1. Take \(W^{[1]}, b^{[1]}, \dots, W^{[n]}, b^{[n]}\) and reshape them into a big vector \(\Theta\)
  2. Take \(dW^{[1]}, db^{[1]}, \dots, dW^{[n]}, db^{[n]}\) and reshape them into a big vector \(d\Theta\)
  3. For each i \(d\Theta_{approx} = \frac{J(\Theta_1, \Theta_2, \dots, \Theta_i + \epsilon, \dots) - J(\Theta_1, \Theta_2, \dots, \Theta_i - \epsilon, \dots)}{2\epsilon}\)
  4. Check the distance 
def gradient_check_n(parameters, gradients, X, Y, epsilon = 1e-7):

    # Set-up variables
    parameters_values, _ = dictionary_to_vector(parameters)
    grad = gradients_to_vector(gradients)
    num_parameters = parameters_values.shape[0]
    J_plus = np.zeros((num_parameters, 1))
    J_minus = np.zeros((num_parameters, 1))
    gradapprox = np.zeros((num_parameters, 1))

    # Compute gradapprox
    for i in range(num_parameters):

        # Compute J_plus[i]. Inputs: "parameters_values, epsilon". Output = "J_plus[i]".
        # "_" is used because the function you have to outputs two parameters but we only care about the first one

        thetaplus = np.copy(parameters_values)
        thetaplus[i][0] = thetaplus[i][0] + epsilon
        J_plus[i], _ = forward_propagation_n(X, Y, vector_to_dictionary(thetaplus))


        # Compute J_minus[i]. Inputs: "parameters_values, epsilon". Output = "J_minus[i]".

        thetaminus = np.copy(parameters_values)
        thetaminus[i][0] = thetaminus[i][0] - epsilon
        J_minus[i], _ = forward_propagation_n(X, Y, vector_to_dictionary(thetaminus))


        # Compute gradapprox[i]

        gradapprox[i] = (J_plus[i] - J_minus[i]) / (2 * epsilon)


    # Compare gradapprox to backward propagation gradients by computing difference.

    numerator = np.linalg.norm(grad - gradapprox)
    denominator = np.linalg.norm(grad) + np.linalg.norm(gradapprox)
    difference = numerator / denominator


    if difference > 2e-7:
        print ("\033[93m" + "There is a mistake in the backward propagation! difference = " + str(difference) + "\033[0m")
    else:
        print ("\033[92m" + "Your backward propagation works perfectly fine! difference = " + str(difference) + "\033[0m")

7 Training Optimization

7.1 Mini-batch gradient descent

Mini-batch gradient descent is a variation of the gradient descent algorithm that splits the training dataset into small batches that are used to calculate model error and update model coefficients. Implementations may choose to sum the gradient over the mini-batch or take the average of the gradient which further reduces the variance of the gradient. Mini-batch gradient descent seeks to find a balance between the robustness of stochastic gradient descent and the efficiency of batch gradient descent. It is the most common implementation of gradient descent used in the field of deep learning.

Using vectorization we can efficiently train on m example, but when this \(m\) value become very big training can become very slow.

  • if m < 10000/20000 we usually use batch gradient descent
  • if m > 10000/20000 we usually use mini-batch gradient descent

Algorithm

  • Having \(X_{(n_x, m)} = [x^{(1)}, x^{(2)}, ..., x^{(m)}]\) and \(Y_{(1, m)} = [y^{(1)}, y^{(2)}, ..., y^{(m)}]\)
  • We split them into mini-batches \(X^{\{1\}} = [x^{(1)}, x^{(2)}, ..., x^{(m/N_of_batches)}]\) and \(Y^{\{1\}} = [y^{(1)}, y^{(2)}, ..., y^{(m/N_of_batches)}]\) for each mini batch.
  • for t = 1, …, nofbatches
    1. Forward propagation on \(X^{\{t \}}\)
      • \(Z^{[1]} = W^{[1]} X^{\{t \}} + b^{[1]}\)

7.2 Exponentially Weighted Averages

Having a list of noisy values we can think about apply moving average algorithm to make the noisy small.

  • Having \(X = x_1, x_2 , ... , x_m\)
  • for each i : \(x_i = \beta x_{i - 1} + (1- \beta) x_i\)
  • each value will be the average of the \(\frac{1}{1-\beta}\) values

(moving average with different values of \(\beta\)

7.2.1 Bias Correction

If the first values are biassed our whole averages would be non-accurate so there is a trick to avoid that

For each \(x_i\) \(x_i = \frac{x_i}{1 - \beta^i}\) so that the first values weight become smaller

7.3 Gradient descent with momentum

Gradient descent sometimes can be slow because for each iteration the weights don't go straight down to the minimum point. We can see that the average of few iteration have a smaller tendence to take bad directions but it is oriented to the minimum so exponentially moving average can help us.

Momentum

  • for each iteration t:
    • Compute \(dw, db\) on current batch/minibatch
    • \(Vdw = \beta Vdw + (1 - \beta) dw\)
    • \(Vdb = \beta Vdb + (1 - \beta) db\)
    • \(w := w - \alpha Vdw\)
    • \(b := b - \alpha Vdb\)

So now we have a new hyperparameter for gradient descent \(\beta\) by default people use \(0.9\)

7.3.1 Python implementation

def update_parameters_with_momentum(parameters, grads, v, beta, learning_rate):

    L = len(parameters) // 2 # number of layers in the neural networks

    # Momentum update for each parameter
    for l in range(L):

        # compute velocities
        v["dW" + str(l+1)] = beta * v["dW" + str(l+1)] + (1 - beta) * grads['dW' + str(l + 1)]
        v["db" + str(l+1)] = beta * v["db" + str(l+1)] + (1 - beta) * grads['db' + str(l + 1)]
        # update parameters
        parameters["W" + str(l+1)] = parameters["W" + str(l+1)] - learning_rate *  v["dW" + str(l+1)]
        parameters["b" + str(l+1)] = parameters["b" + str(l+1)] - learning_rate *  v["db" + str(l+1)]

    return parameters, v

7.4 RMSprop

Algorithm

  • for each iteration t:
    • Compute \(dw, db\) on current batch/minibatch
    • \(Sdw = \beta Sdw + (1 - \beta) dw^2\)
    • \(Sdb = \beta Sdb + (1 - \beta) db^2\)
    • \(w := w - \alpha \frac{dw}{\sqrt{Sdw}}\)
    • \(b := b - \alpha \frac{db}{\sqrt{Sdb}}\)

7.5 Adam Optimization Algorithm

This algorithm put toghether RMSprop and Momentum gradient descent, it is a well know algorithm that the community accepted because good for different application

Algorithm

  • \(Vdw = 0, Ddw=0, Vdb=0, Sdb\)
  • for each iteration t:
    • Compute \(dw, db\) on current batch/minibatch
    • \(Vdw = \beta_1 Vdw + (1 - \beta_1) dw\)
    • \(Vdb = \beta_1 Vdb + (1 - \beta_1) db\)
    • \(Sdw = \beta_2 Sdw + (1 - \beta_2) dw^2\)
    • \(Sdb = \beta_2 Sdb + (1 - \beta_2) db^2\)
    • \(Vdw^{corrected} = \frac{Vdw}{1 - \beta_1^t}\)
    • \(Vdb^{corrected} = \frac{Vdb}{1 - \beta_1^t}\)
    • \(Sdw^{corrected} = \frac{Sdw}{1 - \beta_2^t}\)
    • \(Sdb^{corrected} = \frac{Sdb}{1 - \beta_2^t}\)
    • \(w := w - \alpha \frac{Vdw^{corrected}}{\sqrt{Sdw^{corrected}} + \epsilon}\)
    • \(b := b - \alpha \frac{Vdb^{corrected}}{\sqrt{Sdb^{corrected}} + \epsilon}\)

def update_parameters_with_adam(parameters, grads, v, s, t, learning_rate = 0.01,
                                beta1 = 0.9, beta2 = 0.999,  epsilon = 1e-8):

    L = len(parameters) // 2                 # number of layers in the neural networks
    v_corrected = {}                         # Initializing first moment estimate, python dictionary
    s_corrected = {}                         # Initializing second moment estimate, python dictionary

    # Perform Adam update on all parameters
    for l in range(L):
        # Moving average of the gradients. Inputs: "v, grads, beta1". Output: "v".
        v["dW" + str(l+1)] = beta1 * v["dW" + str(l+1)] + (1 - beta1) * grads['dW' + str(l + 1)]
        v["db" + str(l+1)] = beta1 * v["db" + str(l+1)] + (1 - beta1) * grads['db' + str(l + 1)]

        # Compute bias-corrected first moment estimate. Inputs: "v, beta1, t". Output: "v_corrected".
        v_corrected["dW" + str(l+1)] = (v["dW" + str(l+1)]) / (1 - beta1 ** t)
        v_corrected["db" + str(l+1)] = (v["db" + str(l+1)]) / (1 - beta1 ** t)

        # Moving average of the squared gradients. Inputs: "s, grads, beta2". Output: "s".
        s["dW" + str(l+1)] = beta2 * s["dW" + str(l+1)] + (1 - beta2) * (grads['dW' + str(l + 1)]) ** 2
        s["db" + str(l+1)] = beta2 * s["db" + str(l+1)] + (1 - beta2) * (grads['db' + str(l + 1)]) ** 2

        # Compute bias-corrected second raw moment estimate. Inputs: "s, beta2, t". Output: "s_corrected".
        s_corrected["dW" + str(l+1)] = (s["dW" + str(l+1)]) / (1 - beta2 ** t)
        s_corrected["db" + str(l+1)] = (s["db" + str(l+1)]) / (1 - beta2 ** t)

        # Update parameters. Inputs: "parameters, learning_rate, v_corrected, s_corrected, epsilon". Output: "parameters".
        parameters["W" + str(l+1)] = parameters["W" + str(l+1)] - learning_rate * (v_corrected["dW" + str(l+1)] / (np.sqrt(s_corrected["dW" + str(l+1)]) + epsilon))
        parameters["b" + str(l+1)] = parameters["b" + str(l+1)] - learning_rate * (v_corrected["db" + str(l+1)] / (np.sqrt(s_corrected["db" + str(l+1)]) + epsilon))


    return parameters, v, s

7.8 Batch normalization

The idea beind this technique is to normalize values of \(z^{[l]}\) in the hidden layers

Algorithm

  • For (each) layer
  • Having \(z^{(1)}, \dots, z^{(m)}\)
    • \(\mu = \frac{1}{m} \sum_i z^{(i)}\)
    • \(\sigma^2 = \frac{1}{m} \sum_i (z_i - \mu)^2\)
    • \(z^{(i)}_{norm} = \frac{z^{(i)} - \mu}{\sqrt{\sigma^2 + \epsilon}}\)
    • \(\hat{z}^{(i)} = \gamma z^{(i)}_{norm} + \beta\) <- used instead of z(i)

\(\gamma \: \text{and} \: \beta\) are learnable parameters of the model, they make variance and mean not fixed but choosen

7.8.1 Implementation

\(x\) —— \(w^{[1]}, b^{[1]}\) -—> \(z^[1]\) -—— \(\beta^{[1]}, \gamma^{[1]}\) –——> \(\hat{z}^{[1]}\) —— \(g^{[1]} (\hat{z}^{[1]})\) ——> \(a^{[1]}\)

  • When we use mini-batches this algorithm is applied modularmente to the mini-batch indipendently
  • b parameters become useless, \(\beta\) substitute it

Algorithm

  • for t = 1, …, n of mini-batches
    • Compute forward prop in \(X^{\{ t\}}\)
      • Replace \(z^{[l]}\) with \(\hat{z^{[l]}}\) using batch normalization
    • Use backprop to compute \(dw^{[l]}, d\beta^{[l]}, d\gamma^{[l]}\)
    • Update your parameters

  • Estimate sigma/mu from training using moving average

Choosing different thresholds change our precision/recall.

When the training set is from a different distribution than the development and test sets, the method to analyze bias and variance changes. Having different distribution it can be difficult to enstablish if we have a variance problem or a distribution problem looking at the dev error.

  • Training dev
  • Dev set
  • Test set

A general formulation to evaluate the errors can be the following

  • Bayes error
    • Avoidable bias
  • Training set error
    • Variance
  • Training dev set
    • Data mismatch
  • Dev set
    • Degree of overfitting to the development set
  • Test set
  1. Scenarios examples
    Scenario A

    If the development data comes from the same distribution as the training set, then there is a large variance problem and the algorithm is not generalizing well from the training set. However, since the training data and the development data come from a different distribution, this conclusion cannot be drawn. There isn't necessarily a variance problem. The problem might be that the development set contains images that are more difficult to classify accurately. When the training set, development and test sets distributions are different, two things change at the same time. First of all, the algorithm trained in the training set but not in the development set. Second of all, the distribution of data in the development set is different. It's difficult to know which of these two changes what produces this 9% increase in error between the training set and the development set. To resolve this issue, we define a new subset called training- development set. This new subset has the same distribution as the training set, but it is not used for training the neural network.

    Scenario B

    The error between the training set and the training- development set is 8%. In this case, since the training set and training-development set come from the same distribution, the only difference between them is the neural network sorted the data in the training and not in the training development. The neural network is not generalizing well to data from the same distribution that it hadn't seen before Therefore, we have really a variance problem.

    Scenario C

    In this case, we have a mismatch data problem since the 2 data sets come from different distribution.

    Scenario D

    In this case, the avoidable bias is high since the difference between Bayes error and training error is 10 %.

    Scenario E

    In this case, there are 2 problems. The first one is that the avoidable bias is high since the difference between Bayes error and training error is 10 % and the second one is a data mismatched problem.

    Scenario F

    Development should never be done on the test set. However, the difference between the development set and the test set gives the degree of overfitting to the development set.

In machine learning, a convolutional neural network (CNN, or ConvNet) is a class of deep, feed-forward artificial neural networks, most commonly applied to analyzing visual imagery. CNNs use a variation of multilayer perceptrons designed to require minimal preprocessing. They are also known as shift invariant or space invariant artificial neural networks (SIANN), based on their shared-weights architecture and translation invariance characteristics.

9.1 Vertical edge recognition

Having an image can be helpful detecting edges (vertical or orizontal), and to do that we use the technique called convolution. Convolution is the process of adding each element of the image to its local neighbors, weighted by the kernel. This is related to a form of mathematical convolution. It should be noted that the matrix operation being performed - convolution - is not traditional matrix multiplication, despite being similarly denoted by *.

There are two main types of filters: orizontal and vertical. Orizontal

9.2 Strided Convolution

Stride controls how the filter convolves around the input volume. In the example we had in part 1, the filter convolves around the input volume by shifting one unit at a time. The amount by which the filter shifts is the stride. In that case, the stride was implicitly set at 1. Stride is normally set in a way so that the output volume is an integer and not a fraction.

Normally, programmers will increase the stride if they want receptive fields to overlap less and if they want smaller spatial dimensions.

Having an image you want to detect and localize the presence of one or more objects

Sliding window We start by picking one fixed-size square and use it as conv-net input. We then start to slide it to go through every region of that size. After this step you can change the size of the square to detect different sizes.

Convolution implementation of sliding window

Bounding box prediction YOLO To ge tmore precise bounding box we are going to use YOLO algorithm. The image is divided in \(n\) sections as a grid, we have the convolutional implementation and we can also predict boundary boxes of multiple objects.

NB Bounding boxes values have to be relative to section "cropped" and between 0 and 1.

10.2.1 Evaluating object localization

Intersection over union algorithm is used to evaluate the quality of the prediction in the field of object localization.

Having the actual square and the predicted one we compute the size of the Union and of the Intersection.

\(IoU = \frac{\text{seize of Intersection}}{\text{size of Union}}\)

We can mark it as "correct" if it is > 0.5

10.2.2 Non-max suppression

Sometimes we can get multiple predictions for the same object using convolutional sliding windows. We can use non-max suppression to evaluate the best prediction. We look at the pc value in the output and we select the prediction with the best pc value. We can also discard all the boxes with \(p_c \leq 0.6\) and then apply the filter.

After the process we can discard any remaining box with IoU > 0.5. If not we suppose to have 2 different objects

10.3.1 One shot learning

It is the task of learning to recognise a face having seen one example of that face. This is based on learning a function \(d(img1, img2)\) called similarity function that output a value of similarity between two pictures and having a threshold we can say if the images come from the same person or not.

A good way to to this is to use a Siamese Network Siamese neural network is a class of neural network architectures that contain two or more identical subnetworks. Identical here means they have the same configuration with the same parameters and weights. Parameter updating is mirrored across both subnetworks.

Siamese NNs are popular among tasks that involve finding similarity or a relationship between two comparable things.

Our function of similarity \(d\) would be \(d(x^{(1)}, x^{(2)}) = || f(x^{(1)} - f^{(2)} ||^2\)

Algorithm

  • Parameters of NN define the encoding \(f(x^{(i)})\)
  • Lear parameters so that
    • if \(x^{(i)}, x^{(j)}\) are the same person \(|| f(x^{(i)} - f^{(j)} ||^2\) is small
    • if \(x^{(i)}, x^{(j)}\) are different persons \(|| f(x^{(i)} - f^{(j)} ||^2\) is big

To learn those parameters we are going to use a triplet loss function We have two images in the process of learning, the anchor and the positive/negative image you can train the network by taking an anchor image and comparing it with both a positive sample and a negative sample. The dissimilarity between the anchor image and positive image must low and the dissimilarity between the anchor image and the negative image must be high.

We want \(d(\text{anchor}, \text{positive}) < d(\text{anchor}, \text{negative})\)

We set a new hyperparameter \(\alpha\) the margin paramether used to exclude the non-trivial solutions => \(d(\text{anchor}, \text{positive}) + \alpha < d(\text{anchor}, \text{negative})\)

Triplet loss function Given 3 images : Anchor(A), Negative(N), Positive(P) \(L(A, P , N) = max(||f(A) - f(P)||^2 - ||f(A) - f(N)||^2 + \alpha, 0)\)

Our cost function will be \(J = \sum_{i =0}^m L(A^{(i)},P^{(i)},N^{(i)})\)

Choosing the triplets
if we choose the triplets randomly we can't earn much from it. Insted it is better to select triplets that are hard to train on.

Binary logreg approach We can also see the problem as simple binary classification problem. Having two image we set label to 0 if the two pictures are from different persons and 1 if they came from the same person. \(\hat{y} = \sigma (\sum_k w_i |f(x^{(i)})_k - f(z^{(j)})_k | + b )\)
